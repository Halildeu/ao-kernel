## Final Mekanizma

### Katman ayrımı

| Katman | Sorumluluk | Nerede |
|---|---|---|
| **Backend instantiation** | Instance yarat | Env + Constructor (C+A) |
| **Feature enablement** | enable/strict/fail_action | Policy (`policy_context_memory_tiers.v1.json`) |
| **Capability validation** | Embedding provider/model destek | Registry (mevcut) |

### Fail-closed seçim akışı

```
1. Client arg verildi?          → onu kullan
2. Policy enable=false?         → disabled (no wiring)
3. Env AO_KERNEL_VECTOR_BACKEND?
   - strict=true + fail         → raise (deny)
   - strict=false + fail        → warn + deterministic fallback (mevcut kontrat)
4. Env yok + enable=true        → InMemoryVectorStore (session-scoped)
5. Library mode (.ao/ yok)      → disabled (explicit enable gerek)
```

### Env contract

- `AO_KERNEL_VECTOR_BACKEND=pgvector|inmemory|disabled`
- `AO_KERNEL_PGVECTOR_DSN=postgresql://...`
- `AO_KERNEL_VECTOR_STRICT=0|1`
- `AO_KERNEL_EMBEDDING_PROVIDER=openai|google`
- `AO_KERNEL_EMBEDDING_MODEL=text-embedding-3-small`

### Policy schema extension (minor, backward compat)

```jsonc
{
  "semantic_retrieval": {
    "enabled": true,              // mevcut
    "backend": {
      "type": "pgvector",
      "strict": true,
      "fail_action": "raise"      // "raise" | "warn_fallback"
    },
    "embedding": {
      "provider": "openai",
      "model": "text-embedding-3-small"
    }
  }
}
```
