## Final Decisions (locked for B1 runtime)

### Adapter Contract Evolution

- `agent-adapter-contract.schema.v1.json::capabilities[]` enum +`review_findings` (~5 satır)
- `agent-adapter-contract.schema.v1.json::output_parse[]` rule item shape:
  - Before: `{json_path: str}` (PR-A3 minimal)
  - After: `{json_path: str, capability?: enum_from_capabilities, schema_ref?: str}` — backwards compat
  - `additionalProperties: false` preserved, JSONPath subset `$.key(.key)*` preserved
- `workflow-definition.schema.v1.json::capability_enum` +`review_findings` (drift parity)

### adapter_invoker Extraction Layer

- `_invocation_from_envelope` runs `output_parse` rule walker:
  - Each rule: `json_path` extract → `schema_ref` validate → success: populate `extracted_outputs[capability]` / fail: raise `AdapterOutputParseError`
  - Workflow runtime maps exception to `error.category=output_parse_failed` (workflow-run schema enum, exception has no category field itself)
- `InvocationResult.extracted_outputs: dict[str, dict[str, Any]] = field(default_factory=dict)` — optional field, backwards compat

### Driver/Executor Layer Separation

- `adapter_invoker` = transport + capability-aware extraction (fits existing transport-layer parse pattern)
- `Executor.run_step(driver_managed=True)` = artifact write + step_record CAS (schema-agnostic, PR-A4b pattern preserved)
  - Flow: `result.extracted_outputs[capability]` → `artifacts.write_artifact()` → `step_record.output_ref`

### Edge Case Contracts (B0 pins for B6 runtime)

1. **Multiple `output_parse` rules → same `capability`:** Invalid manifest, fail-closed at manifest load-time (`AdapterManifestCorruptedError`). Rationale: `dict[capability]` carrier cannot have order-dependent ambiguity.
2. **`schema_ref` unresolvable:** Fail-closed — preferred at manifest load-time; worst case at invocation path `AdapterOutputParseError → output_parse_failed`. Rationale: typed payload contract without schema is empty promise.
3. **Extracted payload `null`:** No special prohibition — schema authority (if schema accepts null, OK; if not, `AdapterOutputParseError`). Rationale: null semantics application-specific (e.g., "no findings" valid review outcome).
4. **Envelope has capability field but no `output_parse` rule:** Silently ignored — extraction opt-in. Rationale: manifest-declared extraction prevents warning spam; preserves backwards compatibility for adapters that return extra fields.

1. Docs skeleton (terminology + edge cases + layer separation)
